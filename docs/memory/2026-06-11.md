## 凸轮反转法 滚子接触点+视觉主体+线型 三项修复
- 问题1&2：滚子与凸轮接触点应该随凸轮转动而在滚子表面移动，滚子为主体、杆件弱化
  - 接触点计算：getRollerContactMath(degNow, actualProfile) 直接取 actualProfile[degNow]
  - actualProfile = 凸轮实际轮廓（滚子包络线），profile[degNow] 是滚子圆心，actualProfile[degNow] 是接触点
  - 两者距离精确等于 rr（滚子半径），几何关系正确
  - 删除了畸形的 getRollerContactMathDeg 函数和复杂的 getRollerContactPoint 法向量方法
- 滚子视觉主体化：杆件从4px/2.5px降到1px/1.5px、透明度0.2，滚子边框3px、填充0.2透明
- 坐标变换修正：drawRollerAt 和 drawOscillatingActual 的变换从数学坐标系CCW旋转改为与 ctx.rotate(camAngle) 一致
  - 旧：crx=cp.x*cosA-cp.y*sinA, cry=cp.x*sinA+cp.y*cosA, cx+crx*scale, cy-cry*scale
  - 新：cx+(cp.x*cosA+cp.y*sinA)*scale, cy+(cp.x*sinA-cp.y*cosA)*scale
  - 这使得滚子中心位置在 e=0 时精确为 (cx, cy-totalS*scale)，与简化算法一致
## 凸轮反转法 反转法错位修复 + 概念修正（第二轮）
- 问题1修复：反转法模式完全错位
  - 原因：上次把滚子中心从简化公式(guideX, tipY)改成了getProfilePoint+坐标旋转，导致滚子跟着凸轮旋转而非沿固定导路滑动
  - 修复：回退到简化公式，滚子中心始终在固定导路位置(guideX, tipY)=(cx+e*scale, cy-totalS*scale)
  - 接触点仍用actualProfile[degNow]+camAngle旋转，保证接触点在滚子表面正确移动
  - drawRollerAt签名恢复为(absX, tipY, baseY, degNow, camAngle, actualProfile)
  - drawRollerInLocal签名恢复为(lx, tipY, baseY, degNow, actualProfile)
- 问题2修复：概念修正——滚子从动件凸轮机构的廓线关系
  - 标准理论：理论廓线(pitch curve)=滚子中心轨迹=外侧曲线，实际廓线(cam surface)=理论廓线向内偏移rr=内侧曲线
  - 凸轮轮廓=实际廓线（凸轮物理边界）=内侧曲线
  - 修复：凸轮填充区域从用profile(外侧)做边界改为用actualProfile(内侧)做边界
  - 理论廓线作为单独的参考线画在凸轮外部（黄色），实际廓线作为凸轮边界（橙色）
  - 凸轮填充始终可见（不受checkbox控制），checkbox只控制廓线描边
  - 反转法模式同理：cam fill用actualProfile，理论廓线无填充仅线条
- 问题3：凸轮轮廓实线、基圆虚线
  - drawPartialProfile 中凸轮全轮廓从虚线[4,4]改为实线
  - drawActualMode 中实际廓线（滚子）从 dash:[4,3] 改为实线
  - 基圆已经是虚线[5,4]，无需修改
- 反转法中 drawPartialProfile 的 actualProfile 调用从 computeActualProfile(profile) 改为使用传入的 actualProfile 参数

## 凸轮反转法 第三轮修复 — 基圆位置+凸轮可见性+接触点错位
- 问题1：基圆在凸轮外面（视觉上不对）
  - 原因：第二轮修复把凸轮填充从profile(外侧)改为actualProfile(内侧)，导致基圆(r0)大于实际廓线最小半径(r0-rr)
  - 修复：回退到用profile(外侧/理论廓线)做凸轮填充边界，actualProfile作为内侧参考线
  - 这样基圆(r0)在凸轮填充内部，视觉正确；实际廓线(橙色)在凸轮内部作为参考线
- 问题2：非滚子反转法取消理论廓线勾选后凸轮消失
  - 原因：凸轮填充和廓线高亮都在if(showTheory)内
  - 修复：凸轮填充始终可见(drawPartialProfile始终调用)，checkbox只控制高亮线条颜色
  - showTheory=true时revealedColor='#fbbf24'，showTheory=false时revealedColor=null(不画高亮线)
  - drawPartialProfile增加revealedColor=null检查：if(n>=1 && revealedColor)
- 问题3：滚子反转法取消理论廓线后仍高亮
  - 同问题2的修复，凸轮填充和高亮线条分离控制
- 问题4：滚子反转法接触点错位（乱跑）
  - 根本原因：接触点在ctx.rotate(-degNow)内部绘制，但cam profile在全局坐标系绘制
  - 在局部坐标系中，contactLx=cpMath.x*scale, contactLy=-cpMath.y*scale 经过旋转后映射到全局位置(cx+cpMath.x*scale*cosA-cpMath.y*scale*sinA, ...)，与cam profile全局位置(cx+cpMath.x*scale, cy-cpMath.y*scale)不一致
  - 修复：接触点从drawRollerInLocal移除，改为在ctx.restore()之后全局坐标系绘制
  - 全局坐标：(cx+cpMath.x*scale, cy-cpMath.y*scale) — cam固定不动时直接用math→canvas变换
  - drawRollerInLocal签名简化为(lx, tipY, baseY)，不再需要degNow和actualProfile参数
